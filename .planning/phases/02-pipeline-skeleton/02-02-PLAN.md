From .claude/skills/_shared/rules.md (created in Plan 01):
```markdown
# Shared Pipeline Rules
## CRITICAL RULES (all pipeline skills)
- NEVER use the Write tool on any file inside `.detent/`
- READ state.json at the start of EVERY skill invocation
- VALIDATE that pipeline_stage matches the expected entry state
- WRITE pipeline_stage update at the END of successful completion
- Use `node ./detent-tools.cjs` (relative path)
## Gate Check Pattern
1. Read config: `node ./detent-tools.cjs config-read --dir .`
2. Check config.mode and config.gates.<gate_name>.enabled
3. If supervised AND enabled: AskUserQuestion (proceed/revise/stop)
4. If autonomous OR disabled: proceed without prompting
5. If gates missing: treat as enabled (safe default)
## Skill Exit Pattern
1. state-write --pipeline_stage <this_stage>
2. Print completion summary
3. Print next-step hint
```

Pipeline stage transition table (from RESEARCH.md):
```
idle       -> /detent:discovery -> discovery
discovery  -> /detent:plan      -> planning
planning   -> /detent:code      -> coding
coding     -> /detent:verify    -> verification
verification -> /detent:achieve -> achieve (then complete)
```

Gate placement (from CONTEXT.md decisions):
- Plan gate: in /detent:plan, AFTER discovery output review, BEFORE planning work
- Code gate: in /detent:code, AFTER code generation, BEFORE committing
- Deploy gate: in /detent:achieve, AFTER verification passes, BEFORE merge/deploy

```json
    "code": { "enabled": true, "description": "After code generation - review before commit" },
    "deploy": { "enabled": true, "description": "After verification - review before merge/deploy" }
  }
}
```
