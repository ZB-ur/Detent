**Step 4: Agent Orchestration Loop**

Define the agent sequence and implement a retry-capable loop. The skill must execute these steps using Bash commands. Per D-12, the skill does NOT read agent output files into its own context — it only reads `h-review-verdict.json`.

```
AGENT SEQUENCE (one iteration):
  4a. Spawn D-Critique:
      node ./detent-tools.cjs spawn --dir . --prompt "Run as the d-critique subagent. Read @.detent/truth-surface/frozen-decisions.md @.detent/truth-surface/constraint-ledger.md @.detent/truth-surface/domain-model.md and produce your critique."
      → If exit non-zero: report "D-Critique agent failed. Artifacts preserved in .detent/plan/" and STOP
      → Verify .detent/plan/d-critique-output.md exists

  4b. Spawn G-Red:
      node ./detent-tools.cjs spawn --dir . --prompt "Run as the g-red subagent. Read @.detent/plan/d-critique-output.md @.detent/truth-surface/frozen-decisions.md and produce your attack."
      → If exit non-zero: report "G-Red agent failed" and STOP
      → Verify .detent/plan/g-red-output.md exists

  4c. Spawn G-Blue:
      node ./detent-tools.cjs spawn --dir . --prompt "Run as the g-blue subagent. Read @.detent/plan/g-red-output.md @.detent/plan/d-critique-output.md @.detent/truth-surface/frozen-decisions.md and produce your defense."
      → If exit non-zero: report "G-Blue agent failed" and STOP
      → Verify .detent/plan/g-blue-output.md exists

  4d. Spawn H-Review:
      node ./detent-tools.cjs spawn --dir . --prompt "Run as the h-review subagent. Read @.detent/plan/d-critique-output.md @.detent/plan/g-red-output.md @.detent/plan/g-blue-output.md @.detent/truth-surface/frozen-decisions.md and produce your verdict."
      → If exit non-zero: report "H-Review agent failed" and STOP
      → Verify .detent/plan/h-review-verdict.json exists

  4e. Read H-Review Verdict (ONLY content the skill reads from agent outputs):
      Read .detent/plan/h-review-verdict.json
      Parse JSON: extract `verdict` and `reentry_stage` fields
      → If JSON parse fails: report "H-Review verdict is not valid JSON" and STOP

  4f. Handle Verdict:
      If verdict === "approved": proceed to Step 4g (J-Compile)
      If verdict === "rejected":
        Read current reentry_depth from state
        If reentry_depth >= 2:
          Use AskUserQuestion: "Planning pipeline has been rejected 2+ times. H-Review reason: <reason>. Manual intervention required. Options: retry / stop"
          If retry: reset reentry_depth and continue
          If stop: STOP skill
        Else:
          Increment reentry_depth: node ./detent-tools.cjs state-write --dir . --reentry_depth <current+1>
          If reentry_stage === "D": go back to step 4a
          If reentry_stage === "G": go back to step 4b
          Clear only the output files for stages being re-run (e.g., if reentry to D, clear all .detent/plan/*.md except keep any truth surface changes)

  4g. Spawn J-Compile:
      node ./detent-tools.cjs spawn --dir . --prompt "Run as the j-compile subagent. Read @.detent/plan/d-critique-output.md @.detent/plan/g-red-output.md @.detent/plan/g-blue-output.md @.detent/plan/h-review-verdict.json @.detent/truth-surface/frozen-decisions.md @.detent/truth-surface/domain-model.md and produce the code handoff."
      → If exit non-zero: report "J-Compile agent failed" and STOP
      → Verify .detent/plan/handoff.md exists
```

The skill should include a fallback note: "If agents report they cannot find the referenced files, read the file contents via Bash and embed them directly in the prompt string instead of using @ references."
