# /detent:discovery -- Discovery Stage

Run the Discovery stage: analyze the project scope and produce a domain model artifact. This is a placeholder implementation -- real discovery agents are added in Phase 3+.

## Step 1: Read and Validate State

Read current pipeline state:

```bash
node ./detent-tools.cjs state-read --dir .
```

Parse the JSON output. Verify `pipeline_stage` is `"idle"` OR `"discovery"`.

> Note: Accepting both `idle` and `discovery` is intentional. `idle` is the initial entry point for a fresh pipeline. `discovery` allows re-running discovery without resetting the pipeline -- useful when you want to refresh the domain model after the initial run. This dual-entry is by design, not a bug.

If `pipeline_stage` is neither `"idle"` nor `"discovery"`:
  Stop with message: "Pipeline stage is <actual>, but /detent:discovery expects idle or discovery. Run /detent:<correct_skill> for the current stage instead."

## Step 2: Create Stage Artifacts

Create the discovery output directory and placeholder domain model:

```bash
mkdir -p .detent/discovery
cat > .detent/discovery/domain-model.md << 'ARTIFACT'
# Domain Model

> Placeholder -- real domain model generated by Discovery agent in Phase 3+

Status: pending
ARTIFACT
```

Verify the artifact was created:

```bash
test -f .detent/discovery/domain-model.md && echo "Created" || echo "FAILED"
```

If the output is "FAILED", stop with message: "Failed to create .detent/discovery/domain-model.md"

## Step 3: Update State and Exit

```bash
node ./detent-tools.cjs state-write --dir . --pipeline_stage discovery
```

Print:

```
Discovery complete. Artifacts: .detent/discovery/domain-model.md
Next: /detent:plan
```
