# /detent:verify -- Verification Stage

Run the Verification stage: evaluate the implementation units and produce a verification report. This is a placeholder implementation -- real Evaluator agents are added in Phase 4+.

## Step 1: Read and Validate State

Read current pipeline state:

```bash
node ./detent-tools.cjs state-read --dir .
```

Parse the JSON output. Verify `pipeline_stage` is `"coding"`.

If `pipeline_stage` is not `"coding"`:
  Stop with message: "Pipeline stage is <actual>, but /detent:verify expects coding. Run /detent:code first, then retry /detent:verify."

## Step 2: Create Stage Artifacts

Create the verification output directory and placeholder report:

```bash
mkdir -p .detent/verify
cat > .detent/verify/report.md << 'ARTIFACT'
# Verification Report

> Placeholder -- real report generated by Evaluator agent in Phase 4+

Status: pending
ARTIFACT
```

Verify the artifact was created:

```bash
test -f .detent/verify/report.md && echo "Created" || echo "FAILED"
```

If the output is "FAILED", stop with message: "Failed to create .detent/verify/report.md"

## Step 3: Update State and Exit

```bash
node ./detent-tools.cjs state-write --dir . --pipeline_stage verification
```

Print:

```
Verification complete. Artifacts: .detent/verify/report.md
Next: /detent:achieve
```
